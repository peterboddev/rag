# Medical Claims Data Generator

A Python tool for generating synthetic medical insurance claim datasets for testing and demonstration purposes. This tool combines Synthea-generated synthetic patient records with real TCIA (The Cancer Imaging Archive) medical imaging metadata to produce comprehensive claim datasets.

## Features

- **Synthetic Patient Generation**: Uses Synthea to create realistic FHIR-compliant patient records with cancer diagnoses
- **Real Imaging Integration**: Maps synthetic patients to actual DICOM imaging studies from TCIA
- **Comprehensive Documentation**: Generates CMS-1500 forms, EOB documents, radiology reports, and clinical notes as PDFs
- **Reproducible**: Supports random seed for deterministic data generation
- **Validated Output**: Automatic validation of generated datasets for completeness
- **Statistics**: Generates summary statistics about the dataset
- **S3 Upload**: Optional upload to AWS S3 for cloud storage

## Requirements

### System Requirements

- **Python**: 3.10 or higher
- **Java**: 11 or higher (required for Synthea)
- **Memory**: 2GB+ RAM recommended
- **Disk Space**: 1GB+ for generated data
- **Operating System**: Windows, macOS, or Linux

### External Dependencies

#### 1. Synthea (Required)

Synthea is an open-source synthetic patient generator that creates realistic FHIR-compliant medical records.

**Download and Setup:**
```bash
# Download Synthea
wget https://github.com/synthetichealth/synthea/releases/download/master-branch-latest/synthea-with-dependencies.jar

# Create Synthea directory
mkdir synthea
mv synthea-with-dependencies.jar synthea/

# Test Synthea installation
cd synthea
java -jar synthea-with-dependencies.jar --help
```

**Alternative: Build from source**
```bash
git clone https://github.com/synthetichealth/synthea.git
cd synthea
./gradlew build check test
```

For more information, visit: https://github.com/synthetichealth/synthea

#### 2. TCIA Metadata (Required)

TCIA (The Cancer Imaging Archive) provides real medical imaging metadata that is mapped to synthetic patients.

**Download TCIA Metadata:**

1. Visit: https://www.cancerimagingarchive.net/
2. Navigate to the **EAY131** collection (recommended)
3. Download the metadata CSV file
4. Save as `tcia_metadata.csv` in your working directory

**Required CSV Columns:**
- `PatientID`: Unique patient identifier (e.g., "TCIA-001")
- `StudyInstanceUID`: DICOM study unique identifier
- `Modality`: Imaging modality (e.g., "CT", "MRI")
- `StudyDate`: Date of imaging study (YYYY-MM-DD format)
- `SeriesDescription`: Description of imaging series

**Example CSV format:**
```csv
PatientID,StudyInstanceUID,Modality,StudyDate,SeriesDescription
TCIA-001,1.2.840.113654.2.55.12345,CT,2023-01-15,Chest CT with contrast
TCIA-002,1.2.840.113654.2.55.12346,CT,2023-02-20,Abdomen CT
```

## TCIA Metadata Setup

### CSV Format Requirements

The TCIA metadata CSV must contain the following columns:

| Column | Type | Description | Example |
|--------|------|-------------|---------|
| `PatientID` | String | Unique patient identifier | "TCIA-001" |
| `StudyInstanceUID` | String | DICOM study UID | "1.2.840.113654.2.55.12345" |
| `Modality` | String | Imaging modality | "CT" |
| `StudyDate` | Date | Study date (YYYY-MM-DD) | "2023-01-15" |
| `SeriesDescription` | String | Series description | "Chest CT with contrast" |

### Filtering

The tool automatically filters TCIA metadata for:
- **CT studies only**: Only CT imaging modality is used
- **Valid dates**: Studies with valid date information
- **Complete records**: Records with all required fields

### Minimum Requirements

- At least as many TCIA patients with CT studies as the requested patient count
- For 25 patients (default), need at least 25 TCIA patients with CT studies

## Usage

### Basic Usage

```bash
generate-medical-claims \
    --output-dir ./medical_data \
    --tcia-metadata ./tcia_metadata.csv \
    --patient-count 25 \
    --synthea-path ./synthea
```

### Command-Line Arguments

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `--output-dir` | String | `medical_data` | Root directory for generated data |
| `--tcia-metadata` | String | `tcia_metadata.csv` | Path to TCIA metadata CSV file |
| `--patient-count` | Integer | `25` | Number of patients (20-30) |
| `--seed` | Integer | Random | Random seed for reproducibility |
| `--synthea-path` | String | `./synthea` | Path to Synthea installation |
| `--s3-bucket` | String | None | Optional S3 bucket for upload |
| `--verbose` | Flag | False | Enable verbose (DEBUG) logging |

### Example Commands

**Generate with default settings:**
```bash
generate-medical-claims --tcia-metadata ./tcia_eay131.csv
```

**Generate with specific seed for reproducibility:**
```bash
generate-medical-claims \
    --tcia-metadata ./tcia_eay131.csv \
    --seed 42
```

**Generate maximum patients:**
```bash
generate-medical-claims \
    --tcia-metadata ./tcia_eay131.csv \
    --patient-count 30
```

**Custom output location:**
```bash
generate-medical-claims \
    --output-dir /data/claims \
    --tcia-metadata ./tcia_eay131.csv \
    --synthea-path /opt/synthea
```

**Upload to S3 bucket:**
```bash
generate-medical-claims \
    --tcia-metadata ./tcia_eay131.csv \
    --s3-bucket my-medical-data-bucket
```

**Enable verbose logging:**
```bash
generate-medical-claims \
    --tcia-metadata ./tcia_eay131.csv \
    --verbose
```

### Document Types

#### CMS-1500 Forms
- Standard insurance claim forms
- Include patient demographics, provider information
- CPT codes for CT imaging procedures (71250-71275)
- Reference DICOM Study UID in notes section
- PDF format

#### EOB Documents (Explanation of Benefits)
- Show claim processing results
- **Status Distribution:**
  - 60% Approved (with payment information)
  - 20% Denied (with denial reasons)
  - 20% Pending
- Reference corresponding CMS-1500 claim number
- PDF format

#### Radiology Reports
- Clinical reports for imaging studies
- Include DICOM Study UID
- Imaging modality and anatomical region
- Realistic findings based on cancer diagnosis
- Radiologist name and signature
- PDF format

#### Clinical Notes
- Physician notes for patient encounters
- Patient symptoms and complaints
- Imaging orders (when applicable)
- Treatment plans
- Physician name and signature
- PDF format

## Configuration

### Claim Status Distribution

The tool generates claims with the following status distribution:

- **60% Approved**: Include payment amount and date
- **20% Denied**: Include denial reason code
- **20% Pending**: Awaiting review

### CPT Codes

CT imaging procedures use CPT codes in the range **71250-71275**:

- 71250: CT thorax without contrast
- 71260: CT thorax with contrast
- 71270: CT thorax without and with contrast
- 71275: CT angiography chest

### Cancer Types

The tool generates patients with the following cancer diagnoses:

- **Lung Cancer**: Includes various subtypes and stages
- **Colorectal Cancer**: Includes colon and rectal cancers

## Troubleshooting

### Common Issues

#### Synthea not found

**Error:**
```
Error: Synthea executable not found at ./synthea
FileNotFoundError: Synthea JAR not found: ./synthea/synthea-with-dependencies.jar
```

**Solution:**
- Ensure Synthea is downloaded and extracted
- Verify the JAR file exists: `ls synthea/synthea-with-dependencies.jar`
- Use `--synthea-path` to specify the correct location
- Check Java installation: `java -version`

#### TCIA metadata format error

**Error:**
```
Error: Required columns missing from TCIA metadata
ValueError: TCIA metadata CSV is missing required columns
```

**Solution:**
- Ensure CSV contains all required columns: PatientID, StudyInstanceUID, Modality, StudyDate, SeriesDescription
- Check CSV format and encoding (UTF-8 recommended)
- Verify column names match exactly (case-sensitive)

#### Patient count validation

**Error:**
```
Error: patient_count must be between 20 and 30
```

**Solution:**
- Use `--patient-count` with a value between 20 and 30
- Default is 25 if not specified

#### Insufficient TCIA patients

**Error:**
```
Error: Not enough TCIA patients with CT studies
ValueError: Need at least 25 TCIA patients with CT modality
```

**Solution:**
- Ensure TCIA metadata CSV contains enough patients with CT studies
- Filter metadata to include only CT modality records
- Use a different TCIA collection with more CT studies

#### Java not found

**Error:**
```
Error: Java executable not found
```

**Solution:**
- Install Java 11 or higher
- Verify installation: `java -version`
- Add Java to system PATH

#### Memory issues

**Error:**
```
OutOfMemoryError: Java heap space
```

**Solution:**
- Increase Java heap size by modifying Synthea execution
- Reduce patient count
- Close other applications to free memory

#### S3 upload failures

**Error:**
```
S3 upload failed: Access Denied
```

**Solution:**
- Configure AWS credentials: `aws configure`
- Verify S3 bucket exists and you have write permissions
- Check IAM permissions for S3 PutObject
